# juku

塾システムをAIで構造化するためのリポジトリ。

## 目的

塾運営に必要な学習者モデル・カリキュラム・講師-AI協調・生成AI活用レイヤーを段階的に構築する。

## 設計方針

1. **データモデル先行** — AI化の前に、生徒/講師/教材/セッション履歴/評価のスキーマを固める
2. **講師ダッシュボード優先** — 自動推薦より先に、講師の意思決定支援
3. **学習者モデル (BKT/DKT)** — 単なる正誤ではなく知識状態の確率推定
4. **生成AIは最後** — 土台ができてから対話チューター・解説生成を載せる

## ディレクトリ構成（予定）

```
juku/
├── docs/           # 設計書・ER図・カリキュラム依存グラフ
├── schema/         # データモデル定義
├── packages/
│   ├── core/       # 学習者モデル・推薦エンジン
│   ├── tutor-ui/   # 講師ダッシュボード
│   └── student-ui/ # 生徒向けインターフェース
└── data/           # 教材・問題（gitignore対象）
```
